# Security

This repository is intentionally public and contains only sanitized strategic context.

Do not add:

- credentials, tokens, keys, account identifiers, or private endpoints;
- lead phone lists, private CRM exports, customer data, or family information;
- raw chat logs, private prompts, internal system paths, or unpublished vulnerabilities;
- unsupported revenue, testimonial, or performance claims.

Use public sources and truthful, bounded descriptions. Route sensitive execution through private StewartOS workspaces.
